## Summary

- Flip the default of `--no-prefix-user-supplied-names` to ON (closes #299).
- `cdkd deploy` now skips prepending the stack name to user-declared physical names by default, consistent across every resource type (`roleName: 'my-role'` → AWS resource `my-role`).
- New escape hatch `--prefix-user-supplied-names` (env `CDKD_PREFIX_USER_SUPPLIED_NAMES=true`, cdk.json `context.cdkd.prefixUserSuppliedNames: true`) restores pre-v0.93.0 legacy prefixing on Pattern B providers (IAM Role / User / Group / InstanceProfile / ELBv2 LB / TG).
- Deprecated aliases (`--no-prefix-user-supplied-names`, `CDKD_NO_PREFIX_USER_SUPPLIED_NAMES`, `cdk.json context.cdkd.noPrefixUserSuppliedNames`) still accepted; emit a warning; no-op against the new default.

## Breaking change — minor bump policy

This is a `feat!:` commit. The repo's `.releaserc.json` maps `breaking: true → release: "minor"` because cdkd is in 0.x, so semantic-release bumps 0.92.0 → 0.93.0 (NOT 1.0.0). We stay in v0 while marking the breaking semantics in commit history.

## Migration path

Upgrading from a pre-v0.93.0 cdkd against an existing stack proposes REPLACEMENT on every Pattern B resource with a user-supplied physical name (the AWS resource has the prefixed name; the new template intent has the unprefixed name).

Pick one of:

1. **Accept the one-time replacement** (simplest; safe when affected types tolerate replacement — IAM Roles get fresh ARNs, ELBv2 LBs get fresh DNS names).
2. **Pin legacy prefixing**: pass `--prefix-user-supplied-names`, set `CDKD_PREFIX_USER_SUPPLIED_NAMES=true`, or add `"prefixUserSuppliedNames": true` under `cdk.json` `context.cdkd`.
3. **Drop the explicit physical name** in CDK code where you don't actually need a stable name.

State-side rename helper (`cdkd state rename-strip-prefix <stack>`) that would adopt the new default without replacement is tracked in #300.

## Resolution chain (new)

Highest wins:

1. `--prefix-user-supplied-names` CLI flag → keep legacy prefixing.
2. `CDKD_PREFIX_USER_SUPPLIED_NAMES=true` env var.
3. `cdk.json context.cdkd.prefixUserSuppliedNames: true`.
4. Default `true` (skip prefix — the v0.93.0 flip).

Deprecated aliases (warn + no-op): `--no-prefix-user-supplied-names`, `CDKD_NO_PREFIX_USER_SUPPLIED_NAMES`, `cdk.json context.cdkd.noPrefixUserSuppliedNames`.
